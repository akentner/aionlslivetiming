    ```python
    """nls-replay CLI — replay a recorded JSONL log via the public NLSClient API.

    Per D-03..D-05 + D-25 (see 04-CONTEXT.md). Surfaces:
    - positional: <jsonl_path>
    - --speed N (float, default 1.0; 0 = burst)
    - --limit N (int; default unlimited)
    - --show-time-sync (flag; default off)
    - --strict (flag; default off — raises on UnknownMessage / ReplaySchemaError / ReplayOrderingError)
    - --summary (flag; default off — prints end-of-stream diagnostics)

    Default mode is line-buffered stdout; suitable for piping to `head` / `grep`.
    Exit code is 0 on normal completion (even when WARNING logs are emitted).
    --strict flips this to 1 on any WARNING-level event.
    """

    from __future__ import annotations
    import argparse
    import asyncio
    import collections
    import logging
    import sys
    from pathlib import Path
    from typing import TextIO

    from aionlslivetiming import NLSClient, UnknownEventError, ReplayError
    from aionlslivetiming.events import UnknownMessage, TimeSyncMessage
    from aionlslivetiming.logging import get_logger

    __all__ = ["main", "run"]
    _log = get_logger("aionlslivetiming.cli")

    async def run(
        path: str | Path,
        *,
        speed_factor: float = 1.0,
        limit: int | None = None,
        show_time_sync: bool = False,
        strict: bool = False,
        summary: bool = False,
        out: TextIO | None = None,
    ) -> int:
        """Drive NLSClient.from_replay(path) and print one line per Message.

        Returns 0 on normal completion; 1 if --strict and any error logged at WARNING.
        """
        out = out or sys.stdout
        pid_counts: dict[int, int] = collections.Counter()
        unknown_count = 0
        ts_first_ms: int | None = None
        ts_last_ms: int | None = None
        emitted = 0

        try:
            async with NLSClient.from_replay(
                path, speed_factor=speed_factor
            ) as client:
                # Main messages stream
                async for msg in client.messages():
                    pid = getattr(msg, "event_pid", -1)
                    pid_counts[pid] += 1
                    if isinstance(msg, UnknownMessage):
                        unknown_count += 1
                        _log.warning(
                            "replay: UnknownMessage event_pid=%s raw_keys=%s",
                            pid, list(getattr(msg, "raw", {}).keys()),
                        )
                        if strict:
                            raise UnknownEventError(
                                f"--strict: UnknownMessage at pid={pid}"
                            )
                    out.write(repr(msg) + "\n")
                    emitted += 1
                    if limit is not None and emitted >= limit:
                        break

                # Optional time-sync stream (D-04; ReplayTransport has no time_sync()
                # method, so we read time_sync messages from the underlying JSONL
                # by re-opening it as a line reader — but per D-07 this is not
                # needed in default mode. --show-time-sync is best-effort: emit
                # nothing if the transport suppressed time-sync frames.
                if show_time_sync:
                    async for ts in client.time_sync():
                        out.write(repr(ts) + "\n")

                # Summary block (D-05)
                if summary:
                    ts_first_ms = client.state.last_update_ms  # None if no messages applied
                    ts_last_ms = client.state.last_update_ms
                    # NOTE: ts_recv_ms values come from the JSONL itself, not
                    # the state. For first/last ts, the CLI scans the file once
                    # before constructing the client (D-05 spec).
                    # Implement this by reading ts_recv_ms from each JSONL line
                    # at the start of run() (see below).
                    ...

        except ReplayError as exc:
            _log.warning("replay: %s", exc)
            if strict:
                return 1

        return 0
    ```

    Implementation details to nail down (ALL go in `run()`):

    1. **Pre-scan for first/last ts_recv_ms (D-05):** Before constructing the client, read the JSONL once with `pathlib.Path.open()`. Iterate lines; collect `ts_recv_ms` values (skip lines without one, skip blank lines). Track first and last. This is one extra disk read but acceptable for `--summary` mode. Cache the result and reuse inside the summary block. For non-summary mode, skip this scan entirely.

    2. **`--limit` enforcement:** Use a counter inside the `async for msg in client.messages():` loop. After emitting each Message, check `if limit is not None and emitted >= limit: break`. This breaks out of the iteration; the `async with` exit calls `transport.close()` which is idempotent.

    3. **`--strict` mode (D-25):** Wrap the message iteration in a small observer that catches `UnknownMessage` events on the parsed stream. Default mode: log WARNING + continue. Strict mode: raise `UnknownEventError`. Also catch `ReplaySchemaError` and `ReplayOrderingError` from the transport and re-raise as-is (already typed exceptions). Wrap the WHOLE iteration in a try/except for `ReplayError` family — strict mode returns 1, default mode returns 0.

    4. **`--summary` output (D-05):** After the iteration (or after early break), print to stdout a single block:
       ```
       --- nls-replay summary ---
       total messages: <N>
       per eventPid: {0: 1, 4: 1, 3: 1}
       first ts_recv_ms: <int or 'n/a'>
       last ts_recv_ms: <int or 'n/a'>
       UnknownMessage count: <int>
       ```
       The block is delimited by blank lines so it doesn't get mixed with the message stream when piped.

    5. **`--show-time-sync` mode (D-04):** Since `ReplayTransport` does NOT expose a `time_sync()` method (Pitfall #10; time-sync is suppressed by default), this option has limited utility for replay. Implement it as a best-effort fallback that reads the JSONL a second time and prints any `{type:"time",value:...}` frames as `TimeSyncMessage`. Document this in the help text: "(replay mode only; requires the JSONL to contain time-sync frames)". Actually, per D-16 the JSONL recorder writes time-sync frames into the log but ReplayTransport suppresses them. To surface them, the CLI can scan the raw JSONL for `{"type":"time"...}` patterns. **Implementation:** do a second pass over the JSONL lines, parsing each with `orjson.loads` (or stdlib `json`) and checking for `"type" == "time"`. If found, yield as TimeSyncMessage. This is a documented limitation; users wanting time-sync in replay should write their own consumer.

       **SIMPLER alternative (recommended for this plan):** Just iterate `client.time_sync()` if the transport exposes it (it does not for ReplayTransport), and silently emit nothing. Print a WARNING on stderr if `--show-time-sync` was passed but the transport doesn't support it. This is honest about the limitation and avoids re-implementing parser dispatch in the CLI. Choose this simpler alternative.

       ```python
       parser = argparse.ArgumentParser(

       parser.add_argument("--limit", type=int, default=None,
                           help="Stop after N parsed messages (default: unlimited).")
       parser.add_argument("--show-time-sync", action="store_true", default=False,

       parser.add_argument("--strict", action="store_true", default=False,

       parser.add_argument("--summary", action="store_true", default=False,

       args = parser.parse_args(argv)
       logging.basicConfig(level=logging.INFO, format="%(asctime)s %(levelname)s %(name)s: %(message)s")
       return asyncio.run(run(
           path=args.jsonl_path, speed_factor=args.speed, limit=args.limit,
       ```
